# Security Policy

## Reporting a Vulnerability

Do not open a public issue for a suspected vulnerability or include private user
data, credentials, access tokens, or exploit details in a public discussion.

Report security issues through the repository's **Security** tab using a private
vulnerability report. Include the affected URL, the minimum steps needed to
reproduce the issue, the observed impact, and any relevant request IDs.

Reports are reviewed for the current production branch. Please allow time to
investigate before disclosing an issue publicly.

## Scope

The supported version is the code currently deployed from `main`. Third-party
services such as Supabase, Vercel, GitHub, Google, YouTube, Last.fm, Reddit, and
Groq are governed by their own security and disclosure policies.

## Sensitive Information

Never send passwords, API keys, service-role keys, market-job secrets, database
connection strings, or complete authentication tokens in a report. Redact
personal information that is not required to demonstrate the issue.
